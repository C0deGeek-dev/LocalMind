What is what

LocalMind holds three layers of local knowledge, all reviewed by you. Nothing becomes memory silently.

Review — the candidate queue

Accept & promote: mark the lesson good and write it to durable memory (a Markdown file + a vector embedding) so an agent recalls it. The normal action.

Accept only: mark it accepted but don't write it to memory yet (rare — use if you'll promote later).

Promote: write an already-accepted item to memory (the second half of the pipeline).

Reject: discard the candidate — it will not become memory. Defer: keep it pending for later.

Memory — accepted lessons

Your durable, retrievable lessons. Search or browse, open one for its full text + provenance ("why do you think that"). Delete removes it permanently (recorded in Audit).

Docs — ingested prose

Semantic search and a file browser over every repo's Markdown (READMEs, vision, ADRs, plans). Reference passages, not memory.

Graph — code structure

Symbols and their call / import / test edges. See the overview (counts, languages, hotspots) or search a symbol and walk its neighborhood.

Audit — the log

An append-only record of every action (accept, promote, delete, …). Entries are never edited or deleted — it is the history, not something you change.
